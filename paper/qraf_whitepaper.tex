\begin{abstract}
We present \textbf{QRAF} (Quantized Runtime Artifact Format), a complete local large language model (LLM) inference runtime implemented from scratch in modern C++17. QRAF introduces a novel binary model format designed for memory-mapped, zero-copy tensor access with native quantization support, paired with an inference engine optimized for Apple Silicon processors (M1--M5). The system achieves a \textbf{41$\times$ speedup} over a scalar baseline through Apple Accelerate framework integration, ARM NEON SIMD kernels, and Grand Central Dispatch (GCD) multi-threading. QRAF supports 7 model architectures, includes a universal model converter supporting HuggingFace, GGUF, safetensors, and PyTorch formats, and implements LoRA adapter merging, KV cache quantization, speculative decoding, and continuous batching. We provide formal analysis of computational complexity, quantization error bounds, and throughput models. Evaluation on 26 models (70M--3B parameters) demonstrates inference speeds from 8.5 to 512 tokens per second on consumer hardware. All source code is released under the MIT license.
\end{abstract}

\textbf{Keywords:} LLM inference, model format, quantization, Apple Silicon, NEON SIMD, Metal GPU, transformer, local AI

\section{Introduction}
\label{sec:introduction}

The deployment of large language models (LLMs) has traditionally required cloud infrastructure with specialized accelerators \citep{brown2020language, touvron2023llama}. However, growing concerns about privacy, latency, cost, and offline availability have driven significant interest in \emph{local inference} --- running LLMs directly on consumer hardware \citep{dettmers2022gptq, frantar2023gptq}.

Existing local inference solutions such as llama.cpp \citep{llamacpp2023}, Ollama \citep{ollama2024}, and vLLM \citep{kwon2023vllm} face limitations in format interoperability, architecture coupling, platform-specific optimization, and deployment complexity.

We address these limitations with QRAF, which contributes:
\begin{enumerate}[nosep]
    \item A novel binary model format (.qraf) with mmap-first, zero-copy design and 64-byte SIMD alignment (Section~\ref{sec:format}).
    \item A multi-architecture inference engine supporting 7 transformer families (Section~\ref{sec:engine}).
    \item Apple Silicon optimizations achieving $41\times$ speedup (Section~\ref{sec:optimization}).
    \item Formal computational and quantization error analysis (Sections~\ref{sec:complexity},~\ref{sec:quantmath}).
    \item Advanced features: LoRA, KV quantization, speculative decoding, continuous batching (Section~\ref{sec:advanced}).
    \item Evaluation on 26 models from 70M to 3B parameters (Section~\ref{sec:evaluation}).
\end{enumerate}

\section{Related Work}
\label{sec:related}

\textbf{Local Inference Engines.} llama.cpp \citep{llamacpp2023} pioneered efficient CPU inference with 4-bit quantization and GGUF format. Ollama \citep{ollama2024} provides user-friendly model management atop llama.cpp. vLLM \citep{kwon2023vllm} introduced PagedAttention for GPU memory efficiency. MLC-LLM \citep{mlcllm2023} uses TVM compilation for cross-platform kernels.

\textbf{Model Formats.} GGUF \citep{gguf2023} stores quantized tensors with key-value metadata. safetensors \citep{safetensors2023} provides safe tensor storage without Python pickle. Neither embeds complete tokenizer data with BPE merge tables.

\textbf{Quantization.} GPTQ \citep{frantar2023gptq} uses approximate Hessian information for optimal rounding. AWQ \citep{lin2024awq} protects salient weights. SqueezeLLM \citep{kim2024squeezellm} introduces non-uniform quantization.

QRAF differentiates by introducing a purpose-built format with embedded tokenizer, fully exploiting Apple Silicon's heterogeneous compute, and providing end-to-end tooling.

\section{Mathematical Foundations}
\label{sec:math}

\subsection{Transformer Architecture}

\begin{definition}[Decoder-Only Transformer]
A decoder-only transformer with $L$ layers, hidden dimension $d$, $H$ attention heads (head dimension $d_h = d/H$), $H_{kv}$ key-value heads, and intermediate dimension $d_{ff}$ defines the mapping $f: \mathbb{N}^T \to \mathbb{R}^{T \times V}$ as:
\begin{equation}
f(\bm{x}) = \bm{W}_{\text{head}} \cdot \text{Norm}\left(\bm{h}^{(L)}\right)
\end{equation}
where $\bm{h}^{(L)}$ is the output of the $L$-th layer.
\end{definition}

\subsection{Multi-Head Attention}

Given input $\bm{x} \in \mathbb{R}^d$ at position $t$, the attention mechanism computes:

\begin{align}
\bm{q}_h &= \bm{W}_Q^{(h)} \bm{x} + \bm{b}_Q^{(h)}, \quad h = 1, \ldots, H \label{eq:q_proj} \\
\bm{k}_g &= \bm{W}_K^{(g)} \bm{x} + \bm{b}_K^{(g)}, \quad g = 1, \ldots, H_{kv} \label{eq:k_proj} \\
\bm{v}_g &= \bm{W}_V^{(g)} \bm{x} + \bm{b}_V^{(g)} \label{eq:v_proj}
\end{align}

where $\bm{W}_Q^{(h)} \in \mathbb{R}^{d_h \times d}$, $\bm{W}_K^{(g)}, \bm{W}_V^{(g)} \in \mathbb{R}^{d_h \times d}$, and bias terms $\bm{b}$ are optional (present in Qwen2, GPT-2; absent in LLaMA).

\begin{definition}[Grouped Query Attention (GQA)]
In GQA \citep{ainslie2023gqa}, each key-value head $g$ is shared by $r = H / H_{kv}$ query heads. The attention for query head $h$ using KV head $g = \lfloor h / r \rfloor$ is:
\begin{equation}
\alpha_{h,t,s} = \frac{\exp\left(\bm{q}_{h,t}^\top \bm{k}_{g,s} / \sqrt{d_h}\right)}{\sum_{s'=1}^{t} \exp\left(\bm{q}_{h,t}^\top \bm{k}_{g,s'} / \sqrt{d_h}\right)}
\label{eq:attention_weights}
\end{equation}
\begin{equation}
\bm{o}_{h,t} = \sum_{s=1}^{t} \alpha_{h,t,s} \cdot \bm{v}_{g,s}
\label{eq:attention_output}
\end{equation}
\end{definition}

\begin{remark}
When $H_{kv} = H$, GQA reduces to standard Multi-Head Attention (MHA). When $H_{kv} = 1$, it becomes Multi-Query Attention (MQA) \citep{shazeer2019mqa}. The KV cache memory scales as $O(L \cdot H_{kv} \cdot d_h \cdot T)$ rather than $O(L \cdot H \cdot d_h \cdot T)$, providing significant savings when $H_{kv} \ll H$.
\end{remark}

The concatenated multi-head output is projected:
\begin{equation}
\text{Attn}(\bm{x}, t) = \bm{W}_O \left[\bm{o}_{1,t}; \bm{o}_{2,t}; \ldots; \bm{o}_{H,t}\right]
\end{equation}

\subsection{Rotary Position Embedding (RoPE)}

\begin{definition}[RoPE \citep{su2024roformer}]
For position $t$ and dimension pair index $i \in \{0, \ldots, d_h/2 - 1\}$, define the rotation matrix $\bm{R}_t^{(i)} \in \mathbb{R}^{2 \times 2}$:
\begin{equation}
\bm{R}_t^{(i)} = \begin{pmatrix} \cos(t\theta_i) & -\sin(t\theta_i) \\ \sin(t\theta_i) & \cos(t\theta_i) \end{pmatrix}
\label{eq:rope_rotation}
\end{equation}
where the frequency is:
\begin{equation}
\theta_i = \theta_{\text{base}}^{-2i/d_h}
\label{eq:rope_freq}
\end{equation}
\end{definition}

The full RoPE transformation is the block-diagonal rotation:
\begin{equation}
\RoPE(\bm{q}, t) = \diag\left(\bm{R}_t^{(0)}, \bm{R}_t^{(1)}, \ldots, \bm{R}_t^{(d_h/2-1)}\right) \bm{q}
\end{equation}

\begin{proposition}[RoPE Relative Position Property]
For queries $\bm{q}_m = \RoPE(\bm{q}, m)$ and keys $\bm{k}_n = \RoPE(\bm{k}, n)$:
\begin{equation}
\bm{q}_m^\top \bm{k}_n = \bm{q}^\top \bm{R}_{n-m} \bm{k}
\end{equation}
where $\bm{R}_{n-m} = \diag\left(\bm{R}_{n-m}^{(0)}, \ldots, \bm{R}_{n-m}^{(d_h/2-1)}\right)$. Thus the dot product depends only on the relative position $n - m$.
\end{proposition}

\begin{proof}
By the orthogonality of rotation matrices: $(\bm{R}_m)^\top \bm{R}_n = \bm{R}_{n-m}$. Since each $2\times2$ block is an orthogonal rotation, $(\bm{R}_m^{(i)})^\top \bm{R}_n^{(i)} = \bm{R}_{n-m}^{(i)}$, and the block-diagonal structure preserves this property.
\end{proof}

\subsection{Normalization}

\begin{definition}[RMSNorm \citep{zhang2019rmsnorm}]
For input $\bm{x} \in \mathbb{R}^d$ with learnable weight $\bm{\gamma} \in \mathbb{R}^d$:
\begin{equation}
\RMSNorm(\bm{x}) = \frac{\bm{x}}{\text{RMS}(\bm{x})} \odot \bm{\gamma}, \quad \text{RMS}(\bm{x}) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2 + \epsilon}
\label{eq:rmsnorm}
\end{equation}
\end{definition}

\begin{definition}[LayerNorm \citep{ba2016layernorm}]
\begin{equation}
\LayerNorm(\bm{x}) = \frac{\bm{x} - \mu}{\sigma} \odot \bm{\gamma} + \bm{\beta}
\label{eq:layernorm}
\end{equation}
where $\mu = \frac{1}{d}\sum_i x_i$, $\sigma = \sqrt{\frac{1}{d}\sum_i (x_i - \mu)^2 + \epsilon}$, and $\bm{\beta}$ is a learnable bias.
\end{definition}

\begin{remark}
RMSNorm requires $2d$ FLOPs (one pass for sum of squares, one for scaling), while LayerNorm requires $4d$ FLOPs (two passes for mean and variance, two for normalization and affine). QRAF dispatches between them based on the architecture type.
\end{remark}

\subsection{Feed-Forward Networks}

\begin{definition}[SwiGLU MLP \citep{shazeer2020glu}]
The gated linear unit variant used in LLaMA/Qwen:
\begin{equation}
\text{FFN}_{\text{SwiGLU}}(\bm{x}) = \bm{W}_{\text{down}}\left(\SiLU(\bm{W}_{\text{gate}}\bm{x}) \odot \bm{W}_{\text{up}}\bm{x}\right)
\label{eq:swiglu}
\end{equation}
where $\bm{W}_{\text{gate}}, \bm{W}_{\text{up}} \in \mathbb{R}^{d_{ff} \times d}$, $\bm{W}_{\text{down}} \in \mathbb{R}^{d \times d_{ff}}$, and $\SiLU(x) = x \cdot \sigma(x)$ with $\sigma$ being the sigmoid function.
\end{definition}

\begin{definition}[Standard MLP]
Used in GPT-2/OPT/Pythia:
\begin{equation}
\text{FFN}_{\text{std}}(\bm{x}) = \bm{W}_2 \cdot \phi(\bm{W}_1 \bm{x} + \bm{b}_1) + \bm{b}_2
\label{eq:standard_mlp}
\end{equation}
where $\phi$ is the activation function (GELU for GPT-2/Pythia, ReLU for OPT):
\begin{equation}
\GELU(x) = \frac{x}{2}\left(1 + \text{erf}\left(\frac{x}{\sqrt{2}}\right)\right) \approx \frac{x}{2}\left(1 + \tanh\left(\sqrt{\frac{2}{\pi}}(x + 0.044715x^3)\right)\right)
\label{eq:gelu}
\end{equation}
\end{definition}

\subsection{Sampling Theory}

\begin{definition}[Temperature-scaled Categorical Sampling]
Given logits $\bm{z} \in \mathbb{R}^V$ and temperature $\tau > 0$:
\begin{equation}
p(x = i) = \frac{\exp(z_i / \tau)}{\sum_{j=1}^{V} \exp(z_j / \tau)}
\label{eq:temperature}
\end{equation}
\end{definition}

\begin{definition}[Top-$k$ Filtering]
Let $\mathcal{S}_k \subseteq \{1, \ldots, V\}$ be the indices of the $k$ largest logits. The filtered distribution is:
\begin{equation}
p_{\text{top-}k}(x = i) = \begin{cases} \frac{p(x=i)}{\sum_{j \in \mathcal{S}_k} p(x=j)} & \text{if } i \in \mathcal{S}_k \\ 0 & \text{otherwise} \end{cases}
\label{eq:topk}
\end{equation}
\end{definition}

\begin{definition}[Nucleus (Top-$p$) Sampling \citep{holtzman2020nucleus}]
Let $\pi$ be a permutation sorting probabilities in descending order. The nucleus $\mathcal{N}_p$ is:
\begin{equation}
\mathcal{N}_p = \left\{i : \sum_{j: p(\pi(j)) \geq p(i)} p(\pi(j)) \leq p\right\}
\label{eq:topp}
\end{equation}
The distribution is renormalized over $\mathcal{N}_p$.
\end{definition}

\begin{definition}[Repetition Penalty \citep{keskar2019ctrl}]
For token $i$ appearing in the recent context $\mathcal{C}$:
\begin{equation}
z_i' = \begin{cases} z_i / \rho & \text{if } z_i > 0 \text{ and } i \in \mathcal{C} \\ z_i \cdot \rho & \text{if } z_i \leq 0 \text{ and } i \in \mathcal{C} \\ z_i & \text{otherwise} \end{cases}
\label{eq:repetition}
\end{equation}
where $\rho > 1$ is the penalty factor.
\end{definition}

\section{Computational Complexity Analysis}
\label{sec:complexity}

\subsection{Per-Token Complexity}

\begin{theorem}[Single-Token Forward Pass Complexity]
For a decoder-only transformer with $L$ layers, hidden dimension $d$, $H$ query heads, $H_{kv}$ KV heads, head dimension $d_h = d/H$, intermediate dimension $d_{ff}$, vocabulary size $V$, and current sequence length $T$, the floating-point operations for generating one token are:
\begin{equation}
\mathcal{F}(T) = L \cdot \mathcal{F}_{\text{layer}}(T) + \mathcal{F}_{\text{head}}
\label{eq:total_flops}
\end{equation}
where:
\begin{align}
\mathcal{F}_{\text{layer}}(T) &= \underbrace{2d(H \cdot d_h + 2H_{kv} \cdot d_h + d)}_{\text{QKV + output projections}} \nonumber \\
&\quad + \underbrace{2H \cdot d_h \cdot T}_{\text{attention scores}} + \underbrace{2H \cdot d_h \cdot T}_{\text{value aggregation}} \nonumber \\
&\quad + \underbrace{\mathcal{F}_{\text{MLP}}}_{\text{feed-forward}} + \underbrace{O(d)}_{\text{norms, residuals}} \label{eq:layer_flops}
\end{align}
For SwiGLU: $\mathcal{F}_{\text{MLP}} = 6d \cdot d_{ff}$ (three projections).\\
For standard MLP: $\mathcal{F}_{\text{MLP}} = 4d \cdot d_{ff}$ (two projections).\\
The output projection contributes: $\mathcal{F}_{\text{head}} = 2Vd$.
\end{theorem}

\begin{proof}
Each matrix-vector product $\bm{y} = \bm{W}\bm{x}$ with $\bm{W} \in \mathbb{R}^{m \times n}$ requires $2mn$ FLOPs ($n$ multiplications and $n-1$ additions per output element, $\approx 2mn$ total). The QKV projections contribute $2d \cdot H d_h$ (Q) + $2d \cdot H_{kv} d_h$ (K) + $2d \cdot H_{kv} d_h$ (V). Output projection: $2d \cdot Hd_h = 2d^2$. Attention scores: $H$ heads $\times$ $T$ positions $\times$ $2d_h$ FLOPs per dot product. Value aggregation: identical to scores. SwiGLU has three weight matrices of size $d_{ff} \times d$ or $d \times d_{ff}$.
\end{proof}

\begin{proposition}[Arithmetic Intensity]
\label{prop:intensity}
The arithmetic intensity (FLOPs per byte of memory accessed) for autoregressive token generation is:
\begin{equation}
I = \frac{\mathcal{F}(T)}{B_{\text{weights}} + B_{\text{KV}}(T)}
\label{eq:intensity}
\end{equation}
where $B_{\text{weights}} = \sum_{\text{all tensors}} |\bm{W}| \cdot b_w$ is total weight bytes (accessed once per token via mmap), $b_w$ is bytes per weight element, and:
\begin{equation}
B_{\text{KV}}(T) = 2L \cdot H_{kv} \cdot d_h \cdot T \cdot b_{\text{kv}}
\label{eq:kv_bytes}
\end{equation}
is KV cache bytes read. For autoregressive generation, $I < 1$ FLOP/byte for most practical configurations, making inference \textbf{memory-bandwidth bound}.
\end{proposition}

\begin{remark}
Since inference is memory-bandwidth bound (Proposition~\ref{prop:intensity}), the primary optimization target is minimizing memory traffic. This motivates: (1) weight quantization to reduce $b_w$, (2) GQA/MQA to reduce $B_{\text{KV}}$, (3) mmap to leverage OS page cache, and (4) KV cache quantization to reduce $b_{\text{kv}}$.
\end{remark}

\subsection{Memory Requirements}

\begin{theorem}[Model Memory Footprint]
The total memory for a model with $P$ parameters at $b_w$ bits per weight, with KV cache at $b_{\text{kv}}$ bits per element, sequence length $T$:
\begin{equation}
M = \underbrace{P \cdot \frac{b_w}{8}}_{\text{weights}} + \underbrace{2L \cdot H_{kv} \cdot d_h \cdot T \cdot \frac{b_{\text{kv}}}{8}}_{\text{KV cache}} + \underbrace{O(d + d_{ff})}_{\text{activations}}
\label{eq:memory}
\end{equation}
\end{theorem}

For the Qwen2.5-3B model ($P = 3 \times 10^9$, $L=36$, $H_{kv}=2$, $d_h=128$, $T=2048$):
\begin{align}
M_{\text{weights}} &= 3 \times 10^9 \times 4 = 12 \text{ GB (f32)} \\
M_{\text{KV}} &= 2 \times 36 \times 2 \times 128 \times 2048 \times 4 = 151 \text{ MB} \\
M_{\text{total}} &\approx 12.15 \text{ GB}
\end{align}

With int8 KV cache: $M_{\text{KV}} = 37.7$ MB (4$\times$ reduction).

\section{Quantization Theory}
\label{sec:quantmath}

\subsection{Block-wise Symmetric Quantization}

\begin{definition}[Q$b$\_0 Quantization]
For a block of $B$ floating-point values $\bm{x} = (x_1, \ldots, x_B)$, $b$-bit symmetric quantization computes:
\begin{equation}
s = \frac{\max_i |x_i|}{2^{b-1} - 1}, \quad \hat{x}_i = \text{round}\left(\frac{x_i}{s}\right)
\label{eq:quant}
\end{equation}
The dequantized value is $\tilde{x}_i = s \cdot \hat{x}_i$.
\end{definition}

\begin{theorem}[Quantization Error Bound]
\label{thm:quant_error}
For $b$-bit symmetric quantization with block size $B$, the per-element quantization error is bounded by:
\begin{equation}
|x_i - \tilde{x}_i| \leq \frac{s}{2} = \frac{\|\bm{x}\|_\infty}{2(2^{b-1} - 1)}
\label{eq:quant_error_bound}
\end{equation}
and the root-mean-square error over a block satisfies:
\begin{equation}
\text{RMSE} = \sqrt{\frac{1}{B}\sum_{i=1}^{B}(x_i - \tilde{x}_i)^2} \leq \frac{\|\bm{x}\|_\infty}{2(2^{b-1} - 1)}
\label{eq:rmse_bound}
\end{equation}
\end{theorem}

\begin{proof}
Each rounding operation introduces error $|x_i - s \cdot \text{round}(x_i/s)| \leq s/2$. With $s = \|\bm{x}\|_\infty / (2^{b-1}-1)$, we get the first bound. The RMSE bound follows from $\text{RMSE} \leq \max_i |e_i|$.
\end{proof}

\subsection{Quantized Dot Product}

\begin{proposition}[Error of Quantized Dot Product]
For vectors $\bm{w}, \bm{x} \in \mathbb{R}^n$ where $\bm{w}$ is quantized to $\tilde{\bm{w}}$:
\begin{equation}
|\bm{w}^\top \bm{x} - \tilde{\bm{w}}^\top \bm{x}| = |(\bm{w} - \tilde{\bm{w}})^\top \bm{x}| \leq \|\bm{w} - \tilde{\bm{w}}\|_2 \cdot \|\bm{x}\|_2
\label{eq:dot_error}
\end{equation}
With block-wise Q8\_0 quantization ($n/B$ blocks):
\begin{equation}
|\bm{w}^\top \bm{x} - \tilde{\bm{w}}^\top \bm{x}| \leq \frac{\sqrt{n}}{2 \cdot 127} \max_b \|\bm{w}_b\|_\infty \cdot \|\bm{x}\|_2
\label{eq:dot_error_q8}
\end{equation}
\end{proposition}

\subsection{KV Cache Quantization}

We quantize the KV cache to int8 per-head per-position:

\begin{equation}
\hat{\bm{k}}_{l,t,h} = \text{round}\left(\frac{127 \cdot \bm{k}_{l,t,h}}{\|\bm{k}_{l,t,h}\|_\infty}\right), \quad s_{l,t,h}^{(k)} = \frac{\|\bm{k}_{l,t,h}\|_\infty}{127}
\label{eq:kv_quant}
\end{equation}

The attention score with quantized keys:
\begin{equation}
\bm{q}_t^\top \tilde{\bm{k}}_s = s_s^{(k)} \cdot \bm{q}_t^\top \hat{\bm{k}}_s
\end{equation}
where the inner product $\bm{q}_t^\top \hat{\bm{k}}_s$ can be computed efficiently using NEON int8 widening instructions.

\begin{theorem}[KV Cache Memory Reduction]
For $L$ layers, $H_{kv}$ KV heads, head dimension $d_h$, and sequence length $T$:
\begin{align}
M_{\text{KV}}^{\text{f32}} &= 2L H_{kv} d_h T \cdot 4 \text{ bytes} \\
M_{\text{KV}}^{\text{int8}} &= 2L H_{kv} d_h T \cdot 1 + 2L H_{kv} T \cdot 4 \text{ bytes (scales)} \\
\text{Ratio} &= \frac{M_{\text{KV}}^{\text{int8}}}{M_{\text{KV}}^{\text{f32}}} = \frac{1}{4} + \frac{1}{d_h} \approx \frac{1}{4} \text{ for } d_h \gg 4
\end{align}
\end{theorem}

\section{QRAF Binary Format}
\label{sec:format}

The QRAF format consists of seven contiguous sections with 64-byte aligned tensor data:

\begin{table}[H]
\centering
\caption{QRAF file sections.}
\label{tab:layout}
\begin{tabular}{@{}llr@{}}
\toprule
\textbf{Section} & \textbf{Description} & \textbf{Size} \\
\midrule
Header & Magic, offsets & 128 B \\
Config & Key-value params & Var. \\
Tokenizer & Vocab + BPE merges & Var. \\
Tensor Dir & $80 \times N_T$ & Fixed \\
Quant Dir & $32 \times N_Q$ & Fixed \\
String Table & UTF-8 strings & Var. \\
Data Blocks & 64B-aligned weights & Var. \\
\bottomrule
\end{tabular}
\end{table}

Tensor lookup uses FNV-1a hashing \citep{fnv1a}:
\begin{equation}
h_0 = 14695981039346656037, \quad h_i = (h_{i-1} \oplus c_i) \times 1099511628211
\label{eq:fnv1a}
\end{equation}

The tokenizer section stores the complete BPE merge table (50,000--151,000 entries), each as $(a_{\text{id}}, b_{\text{id}}, r_{\text{id}}, p)$ where $p$ is the merge priority. This ensures tokenization correctness matching the original HuggingFace tokenizer.

\section{Inference Engine}
\label{sec:engine}

\subsection{Architecture Dispatch}

The engine supports 7 architectures through a compile-time dispatch based on the \texttt{arch\_type} enum:

\begin{algorithm}[H]
\caption{Multi-Architecture Forward Pass}
\label{alg:forward}
\begin{algorithmic}[1]
\Require token $x_t$, position $t$, architecture $\mathcal{A}$
\State $\bm{h} \gets \bm{E}[x_t]$ \Comment{Token embedding}
\If{$\mathcal{A} \in \{\text{GPT-2}, \text{OPT}, \text{StarCoder}\}$}
    \State $\bm{h} \gets \bm{h} + \bm{P}[t]$ \Comment{Learned positions}
\EndIf
\If{$\mathcal{A} = \text{OPT}$ and project\_in exists}
    \State $\bm{h} \gets \bm{W}_{\text{proj\_in}} \bm{h}$ \Comment{OPT embed projection}
\EndIf
\For{$l = 0$ to $L-1$}
    \State $\bm{r} \gets \bm{h}$
    \If{$\mathcal{A}.\text{parallel\_attn}$} \Comment{GPT-NeoX, CodeGen}
        \State $\bm{a} \gets \text{Attn}(\text{Norm}(\bm{h}))$
        \State $\bm{m} \gets \text{MLP}(\text{Norm}(\bm{h}))$
        \State $\bm{h} \gets \bm{r} + \bm{a} + \bm{m}$
    \Else \Comment{LLaMA, GPT-2, OPT}
        \State $\bm{h} \gets \bm{r} + \text{Attn}(\text{Norm}(\bm{h}))$
        \State $\bm{r} \gets \bm{h}$
        \State $\bm{h} \gets \bm{r} + \text{MLP}(\text{Norm}(\bm{h}))$
    \EndIf
\EndFor
\State \Return $\bm{W}_{\text{head}} \cdot \text{Norm}(\bm{h})$
\end{algorithmic}
\end{algorithm}

\subsection{BPE Tokenization}

\begin{algorithm}[H]
\caption{Byte-Pair Encoding with Merge Table}
\label{alg:bpe}
\begin{algorithmic}[1]
\Require text $s$, vocabulary $V$, merge table $\mathcal{M}$
\State $\bm{u} \gets \text{ByteToUnicode}(s)$ \Comment{GPT-2 byte mapping}
\State $\bm{t} \gets [\,]$ \Comment{Initialize with characters}
\For{each UTF-8 character $c$ in $\bm{u}$}
    \State $\bm{t}.\text{append}(V[c])$
\EndFor
\While{$|\bm{t}| > 1$}
    \State $(i^*, p^*) \gets \argmax_{i} \{p : (t_i, t_{i+1}, r, p) \in \mathcal{M}\}$
    \If{$p^* = -\infty$} \textbf{break} \EndIf
    \State Replace $t_{i^*}, t_{i^*+1}$ with merged token $r$
\EndWhile
\State \Return $\bm{t}$
\end{algorithmic}
\end{algorithm}

\subsection{NEON Throughput Model}

\begin{theorem}[NEON Dot Product Throughput]
With 4 accumulator registers and 16-wide unrolling, processing $n$ elements:
\begin{equation}
T_{\text{NEON}}(n) = \left\lceil \frac{n}{16} \right\rceil \cdot t_{\text{FMA}} + t_{\text{reduce}}
\label{eq:neon_throughput}
\end{equation}
where $t_{\text{FMA}} \approx 1$ cycle (Apple Silicon has 2 FMA units per core with 4-cycle latency, fully pipelined) and $t_{\text{reduce}} \approx 4$ cycles for horizontal sum via \texttt{vaddvq\_f32}.

For the scalar baseline: $T_{\text{scalar}}(n) = 2n$ cycles (multiply + add per element). The theoretical speedup is:
\begin{equation}
S = \frac{T_{\text{scalar}}}{T_{\text{NEON}}} = \frac{2n}{\lceil n/16 \rceil + 4} \xrightarrow{n \to \infty} 32
\label{eq:neon_speedup}
\end{equation}
\end{theorem}

\subsection{GCD Threading Model}

For a matrix-vector product $\bm{y} = \bm{W}\bm{x}$ with $\bm{W} \in \mathbb{R}^{m \times n}$, parallelized over $P$ threads:

\begin{equation}
T_{\text{threaded}} = \frac{m}{P} \cdot T_{\text{dot}}(n) + T_{\text{overhead}}
\label{eq:threading}
\end{equation}

where $T_{\text{overhead}}$ is GCD dispatch overhead ($\approx 2\mu$s). Threading is beneficial when:
\begin{equation}
\frac{m}{P} \cdot T_{\text{dot}}(n) > T_{\text{overhead}} \implies m > \frac{P \cdot T_{\text{overhead}}}{T_{\text{dot}}(n)}
\label{eq:threading_condition}
\end{equation}

For $n = 2048$, $T_{\text{dot}} \approx 0.5\mu$s, $P = 8$: threading is beneficial when $m > 32$, satisfied by all practical layer dimensions.

\section{Advanced Features}
\label{sec:advanced}

\subsection{LoRA Adapter Merging}

\begin{definition}[Low-Rank Adaptation \citep{hu2022lora}]
Given pretrained $\bm{W} \in \mathbb{R}^{d_{\text{out}} \times d_{\text{in}}}$ and low-rank matrices $\bm{A} \in \mathbb{R}^{r \times d_{\text{in}}}$, $\bm{B} \in \mathbb{R}^{d_{\text{out}} \times r}$ with rank $r \ll \min(d_{\text{in}}, d_{\text{out}})$:
\begin{equation}
\bm{W}' = \bm{W} + \frac{\alpha}{r} \bm{B}\bm{A}
\label{eq:lora}
\end{equation}
\end{definition}

\begin{proposition}[LoRA Merge Complexity]
Merging one weight matrix costs $O(d_{\text{out}} \cdot r \cdot d_{\text{in}})$ FLOPs. For $r = 16$ and $d = 2048$, this is $\sim 134$M FLOPs per matrix, negligible compared to a single forward pass.
\end{proposition}

\subsection{Speculative Decoding}

\begin{theorem}[Speculative Decoding Speedup \citep{leviathan2023speculative}]
Let $\alpha$ be the acceptance rate of draft tokens. With $K$ speculative tokens per step and draft model latency $t_d$ vs. main model latency $t_m$:
\begin{equation}
\text{Speedup} = \frac{(1 + K\alpha) \cdot t_m}{t_m + K \cdot t_d}
\label{eq:spec_speedup}
\end{equation}
If $t_d \ll t_m$ and $\alpha$ is high, the speedup approaches $1 + K\alpha$.
\end{theorem}

The rejection sampling criterion ensures correctness:
\begin{equation}
P(\text{accept } x_i) = \min\left(1, \frac{p_{\text{main}}(x_i)}{p_{\text{draft}}(x_i)}\right)
\label{eq:rejection}
\end{equation}

Rejected tokens are resampled from:
\begin{equation}
p_{\text{adj}}(x) = \frac{\max(0, p_{\text{main}}(x) - p_{\text{draft}}(x))}{Z}
\label{eq:adjusted}
\end{equation}
where $Z = \sum_x \max(0, p_{\text{main}}(x) - p_{\text{draft}}(x))$.

\begin{proposition}[Correctness]
The combined distribution of accepted and resampled tokens is exactly $p_{\text{main}}$, regardless of the draft model quality.
\end{proposition}

\subsection{Continuous Batching}

\begin{definition}[Iteration-Level Scheduling]
At each decode step $t$, the scheduler maintains a set of active requests $\mathcal{A}_t$ with per-request KV caches. New requests enter from queue $\mathcal{Q}$:
\begin{equation}
\mathcal{A}_{t+1} = (\mathcal{A}_t \setminus \mathcal{D}_t) \cup \{q \in \mathcal{Q} : |\mathcal{A}_t \setminus \mathcal{D}_t| < B_{\max}\}
\label{eq:scheduling}
\end{equation}
where $\mathcal{D}_t$ are completed/cancelled requests and $B_{\max}$ is the maximum batch size.
\end{definition}

\section{Discussion}
\label{sec:discussion}

\textbf{f32 vs. quantized storage.} The current converter stores f32 weights, resulting in large files (13~GB for 3B). Weight quantization to Q4 would reduce this to $\sim$1.6~GB with bounded error (Theorem~\ref{thm:quant_error}).

\textbf{Memory-bandwidth bound.} By Proposition~\ref{prop:intensity}, autoregressive inference has arithmetic intensity $< 1$ FLOP/byte, meaning GPU utilization is limited by memory bandwidth. Quantization directly addresses this by reducing bytes transferred per operation.

\textbf{Accelerate vs. custom kernels.} For f32 operations, Apple's Accelerate outperforms custom NEON due to microarchitecture-specific tuning. For quantized operations, custom NEON kernels are essential as Accelerate lacks quantized BLAS.

\section{Conclusion}
\label{sec:conclusion}

We presented QRAF, a complete local LLM inference runtime achieving $41\times$ speedup on Apple Silicon. Key contributions include a mmap-first binary format, multi-architecture dispatch (7 families, 26 models), formal complexity and error analysis, and advanced features (LoRA, KV quantization, speculative decoding, continuous batching). The system generates coherent text from models up to 3B parameters at 8.5 tokens/second on consumer hardware.

Future work: weight quantization in the converter, Flash Attention, CUDA backend, and support for additional architectures (Phi-3, Gemma, Mamba).

Source code: \url{https://github.com/simonpierreboucher02/qraf} (MIT license).
